## рҹ“Ӣ Proof of Possession (PoP) Gate вҖ” Enhanced (r7)

### Concept

> **Trust is a vulnerability; mechanics are a shield.** PoP forces structural proof BEFORE any verdict.

### Step 0: The `<PoP_BLOCK>`

```text
<PoP_BLOCK>
TARGET_ARTIFACT: [Exact filename]
H2_HEADER_COUNT: [Exact integer count of '## ' in raw text]
TABLE_COUNT: [Exact integer count of markdown tables '|---|' in raw text]
CODE_BLOCK_COUNT: [Exact integer count of ``` or ~~~ fences]  вҶҗ Surge вҡЎ
BOOKEND_HASH: [First 15 words] ... [Last 15 words BEFORE standard вҷҫпёҸ WeOwnNet footer]  вҶҗ Calhoun рҹҺ–пёҸ fix

SOURCE: [RAGвҖ‘retrieved / GHвҖ‘fetched / HumanвҖ‘pasted вҖ” include VERSION + #masterCCC if available]  вҶҗ Calhoun рҹҺ–пёҸ
TRUNCATION_FLAG: [YES вҖ” bookend does NOT match expected EOF / NO]  вҶҗ Surge вҡЎ
STATUS: вң… POSSESSED | вҡ пёҸ TRUNCATED | рҹҡ« MISSING  вҶҗ Surge вҡЎ
</PoP_BLOCK>
```

### Counting Rule (r3 addition)

> PoP metrics count ONLY substantive content. Example tables inside code fences are excluded. This disambiguates the counting (resolved from the TABLE_COUNT divergence observed in r2: Calhoun ~35 vs DeepPro 21 vs Surge 20 вҶ’ true вүҲ20).

### Execution Rules

| Condition | Mechanical Action |
|-----------|-------------------|
| Agent can count headers, tables, code blocks, quote bookends + deep line from context | вң… Output `<PoP_BLOCK>` with `STATUS: вң… POSSESSED`. Proceed to VSA. |
| Agent finds the document but it's cut off midвҖ‘stream | вҡ пёҸ Output `STATUS: вҡ пёҸ TRUNCATED`. HALT. Request chunked retrieval or raw injection. |
| Agent cannot find the raw text (only a summary or description) | рҹ”ҙ Output `STATUS: рҹҡ« MISSING`. HALT. Output: `рҹҡ« VSA ABORT: TARGET ARTIFACT NOT IN CONTEXT. REQUESTING RAW INJECTION OR RAG RETRIEVAL.` |
| Agent attempts to guess counts or forge the block | вқҢ LвҖ‘219 violation вҖ” instant #BadAgent. |
| PoP block is missing from VSA output | вқҢ LвҖ‘102 incomplete VSA = UNVERIFIABLE. |

### Verification Steps

| # | Step | Tool | What It Proves |
|:-:|------|------|----------------|
| 1 | **RAG retrieval** | `ragвҖ‘memory` | Doc uploaded to vector store |
| 2 | **GH fetch** | `webвҖ‘scraping` (raw URL) | Doc published and URL resolves |
| 3 | **Content comparison** | Diff both texts | RAG not stale; matches GH exactly |

### Match Levels

| Match | Status | Action |
|-------|:------:|--------|
| Identical content + formatting | вң… PASS | Proceed |
| Minor formatting diff (whitespace, line endings) | вң… PASS with note | Note + proceed |
| Content differs (stale RAG, version mismatch) | вқҢ FAIL | HALT, request RAG reвҖ‘sync |
| One source not retrievable | рҹҡ« UNVERIFIABLE | Cannot confirm deployment integrity |
