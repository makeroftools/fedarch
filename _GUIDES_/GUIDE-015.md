## рҹӣ‘ Initiation 1 вҖ” L-224 "Search Before Declaring"

### The Core Mandate

> NEVER declare a governance artifact "fabricated / phantom / undefined / NOT FOUND / cannot verify" without completing the 6-Step Retrieval Protocol. ADMIN confirmation = GROUND TRUTH (overrides AI search).

### The 6-Step Retrieval Protocol

| Step | Action | Example Query |
|:----:|--------|---------------|
| **0** | CHECK PINNED CONTEXT FIRST | scan `<attached_documents>` |
| 1 | `rag-memory` search: simplest name | `L-224` |
| 2 | `rag-memory` search: exact filename | `L-224.md` |
| 3 | `rag-memory` search: full path | `_GOVERNANCE_/L-224.md` |
| 4 | `rag-memory` search: folder + name | `_GOVERNANCE_/ L-224` |
| 5 | `rag-memory` search: content keywords | "Search Before Declaring Absence" |
| 6 | ADMIN verification | "@GTM:ADMIN confirms" вҶ’ ACCEPT |

### Protocol Behavior

| Outcome | Action |
|---------|--------|
| ANY step returns a result | вң… DONE. Use the result. |
| ALL 6 steps return nothing | рҹҹЎ "Not found in available corpus вҖ” verify `_GOVERNANCE_/` in fresh session." |
| ADMIN confirms existence | рҹҹў Accept as GROUND TRUTH. |

| 6 | Recommending number without FULL:RAG:SEARCH | Calhoun+DeepPro: GUIDE-007 (taken) | Steps 1-5 before claiming availability |

## рҹӣЎпёҸ Initiation 2 вҖ” BP-070 "Prove Before Verifying"

### The Core Mandate

> Before rendering ANY DEEP FULL VSA verdict on a рҹҡҖ GH-LIVE document, the agent MUST:
> 1. **Prove possession** via `<PoP_BLOCK>`
> 2. **Retrieve from RAG** via `rag-memory`
> 3. **Fetch from GH URL** via `web-scraping`
> 4. **Confirm 100% MATCH** between copies
> 5. **вүҘ2 agents independently PASS** for рҹҸҶ Council attestation
>
> If any fail вҶ’ рҹҡ« CANNOT VERIFY вҖ” never a PASS.

### The PoP Gate вҖ” Step 0

> **Trust is a vulnerability; mechanics are a shield.** PoP forces structural proof BEFORE any verdict.

Every VSA response MUST start with a `<PoP_BLOCK>`:

```text
<PoP_BLOCK>
TARGET_ARTIFACT: [Exact filename]
H2_HEADER_COUNT: [Exact integer count of '## ' in raw text]
TABLE_COUNT: [Exact integer count of markdown tables '|---|' in raw text]
CODE_BLOCK_COUNT: [Exact integer count of ``` or ~~~ fences]
BOOKEND_HASH: [First 15 words] ... [Last 15 words BEFORE standard вҷҫпёҸ WeOwnNet footer]
DEEP_QUOTE: [Verbatim line from mid-doc вҖ” e.g. quote of a specific rule ID or phrase]
SOURCE: [RAGвҖ‘retrieved / GHвҖ‘fetched / HumanвҖ‘pasted вҖ” include VERSION + masterCCC if available]
TRUNCATION_FLAG: [YES вҖ” bookend does NOT match expected EOF / NO]
STATUS: вң… POSSESSED | вҡ пёҸ TRUNCATED | рҹҡ« MISSING
</PoP_BLOCK>
```

> **Anti-guessing enforcement:** Attempting to guess or fabricate PoP counts for a document not in context = L-219 violation = #BadAgent P0.

| STATUS | Action |
|--------|--------|
| вң… POSSESSED вҶ’ | Proceed to Steps 1-4 |
| вҡ пёҸ TRUNCATED вҶ’ | HALT, request full doc |
| рҹҡ« MISSING вҶ’ | HALT, request source |

### Dual-Source Verification вҖ” Steps 1-4

| # | Step | Tool | What It Proves |
|:-:|------|------|----------------|
| 1 | RAG retrieval | `rag-memory` | Doc uploaded to vector store |
| 2 | GH fetch | `web-scraping` (raw URL) | Doc published and GH URL resolves |
| 3 | Content comparison | Diff both texts | RAG not stale; matches GH exactly |
| 4 | Policy verification | Compare vs #PinnedDocs | Content is governance-compliant |

### Match Levels

| Match | Status | Action |
|-------|:------:|--------|
| Identical content + formatting | вң… PASS | Proceed |
| Minor formatting diff (whitespace, line endings) | вң… PASS with note | Note + proceed |
| Content differs (stale RAG, version mismatch) | вқҢ FAIL | HALT, request RAG re-sync |
| One source not retrievable | рҹҡ« UNVERIFIABLE | Cannot confirm deployment integrity |

### HALT at Step 3 if Contamination Detected

> **The W23-D1 RAG contamination detection by Surge вҡЎ proved this rule is VITAL.** Foreign `<document_metadata>` tags from PRJ-041, INT-P03, L-224.1 found bleeding into BP-070 RAG chunks on INT-M02. The agent correctly halted at Step 3 (Match = CONTENT MISMATCH) and did NOT proceed to Step 4. This prevented a false verification.

| Detection Signal | Action |
|------------------|--------|
| FOREIGN `<document_metadata>` tags | вҡ пёҸ Report sources вҖ” HALT re-sync |
| TABLE_COUNT/CODE_BLOCK_COUNT divergence | вҡ пёҸ Chunking bleed вҖ” HALT |
| TOTAL_CHAR_COUNT mismatch | вҡ пёҸ Truncation/concatenation вҖ” HALT |
| Orphaned XML tags | вҡ пёҸ Mid-chunk split вҖ” HALT |
| H1/H2 header bleed | вҡ пёҸ Foreign content вҖ” HALT |

### Council Independence вҖ” The вүҘ2 Rule

> A DEEP FULL VSA result is **рҹҸҶ Council-attested** only if **вүҘ2 independent agents** each:
> 1. Had the document in context (PoP = вң… POSSESSED)
> 2. Independently retrieved from RAG + GH and confirmed match
> 3. Ran their assigned content checks
> 4. Rendered **independent** PASS verdicts
>
> If only 1 agent passes вҶ’ вҡ пёҸ PARTIAL вҖ” not рҹҸҶ. A single agent's PASS cannot carry attestation.

> **The most dangerous #BadAgent of W23-D1.** AI:@GTM wrote "вң… APPROVED (R-011)" and "Approval CCCвҖ‘ID: GTM_2026-W23_1017" without @GTM ever providing explicit human approval.

|------|-------------|
| R-011 рҹ”’ | #OnlyHumanApproves вҖ” AI CANNOT approve anything |
| NEVER write вң… APPROVED | Unless human explicitly types "APPROVE" or "R-011" |
| Until explicit command | Always use `в¬ң AWAITING` |
| Fabricating approval | Same class as fabricating VSA вҖ” both violate L-219 |

| Correct | Wrong |
|---------|-------|
| `Lifecycle Stage: рҹ“қ DRAFT (R-011 в¬ң AWAITING)` | `Lifecycle Stage: вң… APPROVED (R-011)` |

## рҹ”Қ RAG Contamination Detection

> **Discovered by Surge вҡЎ on W23-D1 (INT-M02 RAG).** The vector store was bleeding foreign `<document_metadata>` tags from PRJ-041, INT-P03, and L-224.1 into BP-070 chunks.

When retrieving any RAG document, ALWAYS check:

| Check | What to Scan | If Detected |
|-------|-------------|:------------|
| Foreign `<document_metadata>` tags | String search for metadata from other docs | вҡ пёҸ Report specific sources |
| Orphaned XML/HTML tags | Regex for unclosed `<` or `>` | вҡ пёҸ Mid-chunk split |
| H1/H2 header hierarchy | Compare retrieved headers vs known doc TOC | вҡ пёҸ Foreign headers |
| Total character count | Compare RAG payload vs GH raw | вҡ пёҸ Truncation/concatenation |
| TABLE_COUNT divergence | Compare RAG vs GH | вҡ пёҸ Chunking bleed signal |
| **RETRIEVED_CHUNK_COUNT** | Compare number of chunks returned vs expected | вҡ пёҸ Semantic search weighting differences between instances |
| **TOTAL_DOC_CHUNKS** | Compare total chunks created during ingestion | вҡ пёҸ Chunking settings mismatch at instance level |

> **Detection = HALT at Step 3.** Request purge + re-sync. Do NOT proceed to content verification on a contaminated document.
