| Directive | Implementation | Source |

## 🛡️ Phase 0: The Security Gate (Wordfence Freeze)

> **Applies to:** 🔥 **ALL WordPress deployments. Flagship first: BurnedOut.xyz**
> **Source:** DeepPro 🌊

| Rule | Detail |
|------|--------|
| 🛑 **PLUGIN FREEZE** | **DO NOT** install Koko, Performance Lab, Query Monitor, or any optimization plugins while the Wordfence Care audit is active |
| **Rationale** | New code introduces unfamiliar signatures to the malware scanner, potentially triggering false positives or masking actual vulnerabilities |
| **Clearance** | Phase 1 begins ONLY after Wordfence Care delivers the final report AND all critical/high findings are resolved |

## 📊 Phase 1: Traffic Analytics & Matomo MCP

> **Applies to:** 🔥 **ALL WordPress deployments. Flagship first: BurnedOut.xyz**
> **Dual-Stack Approach** — Matomo for AI agents (via MCP) + Koko for human WP Admin UI
> **Matomo is inherently multi-tenant — adding a new site = install Matomo Analytics for WP plugin + point to existing `matomo.WeOwn.app` + Matomo assigns new `idsite`. MCP bridge scales across ALL sites with zero new infrastructure.**
> **Source:** Surge ⚡ + Calhoun 🎖️

### The Dual Stack

| Layer | Tool | Purpose | AI/MCP Visibility |
|-------|------|---------|:-----------------:|
| **System of Record (AI)** | **Matomo** (self-hosted, GPLv3) | Deep analytics, goal tracking, privacy-first. **Already deployed** for flagship. New sites add to same instance. | ✅ **FULL** (via MCP) |
| **MCP Bridge** | **`t-matomo_tool`** | Wraps Matomo's HTTP Reporting API. Allows #MetaCouncil to query traffic natively. | ✅ **FULL** |
| **WP Admin UI (Human)** | **Koko Analytics** (GPLv2, ~3KB) | Cookie-free, zero external requests. Quick pageview checks in WP Admin. | ❌ **NONE** |
| **WP Integration** | **Matomo Analytics for WP** | Embeds Matomo dashboard in WP Admin sidebar. | ✅ UI only |

### Matomo MCP Build Path (Standard)

| Step | Action | Rule |
|:----:|--------|------|
| 1 | Generate read-only `token_auth` in Matomo Admin → Platform → API | — |
| 2 | Deploy `matomo-mcp-server` (Node.js or Python) on INT-P01 or INT-M02 | — |
| 3 | Register Tool Agent `t-matomo_tool` | R-198 |
| 4 | Create MAIT_Matomo @ INT-P01:tools | R-176, L-093 |
| 5 | Register MCP server in WeOwnLLM `@agent` router config | — |
| 6 | For each new WP site: install Matomo Analytics for WP plugin → point to `matomo.WeOwn.app` → note `idsite` | ∞ **scales infinitely** |

## ⚡ Phase 2: Technical Speed & Health (Core Web Vitals)

> **Applies to:** 🔥 **ALL WordPress deployments. Flagship first: BurnedOut.xyz**
> **Source:** Calhoun 🎖️, DeepPro 🌊, Surge ⚡

### Tier 1: Real User Monitoring (Continuous)

| Tool | License | Cost | Measures | Installation |
|------|:-------:|:----:|:---------|:------------|
| **Performance Lab** (Web Vitals module) | GPLv2 | $0 | LCP, CLS, INP from real visitors. Official WP Performance Team plugin. | Plugin → **Enable ONLY Web Vitals module.** |

### Tier 2: Diagnostic "X-Ray" Stack (Admin-Only — Deactivate After Use)

| Tool | License | Cost | Focus |
|------|:-------:|:----:|-------|
| **Query Monitor** | GPLv2 | $0 | DB queries, PHP errors, HTTP API calls, memory. Identifies exactly WHICH plugin is slow. |
| **Health Check & Troubleshooting** | GPLv2 | $0 | Server config audit. Troubleshooting Mode disables plugins for YOUR user only. |
| **WP Crontrol** | GPLv2 | $0 | Audit stuck/orphaned cron jobs — hidden cause of CPU spikes. |

### Tier 3: Server Infrastructure

| Tool | License | Cost | Measures | Deployment |
|------|:-------:|:----:|:---------|:----------:|
| **Netdata** | GPLv3 | $0 | CPU, RAM, disk, MySQL, network | One instance monitors ALL WP deployments |
| **Redis Object Cache** | GPLv2 | $0 | Offloads DB queries from PHP to RAM (80%+ reduction) | Per-server deployment |

### Surge ⚡ Diagnostic Protocol (Live Site Safety)

| Phase | Action | Safety Guardrail |
|:-----:|--------|------------------|
| 1 | Enable **Health Check** — fix Critical warnings | First — before anything |
| 2 | **Troubleshooting Mode** — plugins disabled ONLY for your session | Does NOT affect live visitors |
| 3 | Activate **Query Monitor** — browse slowest pages | Look at "Queries by Component" |
| 4 | Audit **WP Crontrol** — delete stuck/orphaned cron jobs | Hidden CPU/DB drain |
| 5 | Clean & **Deactivate** — remove transients, deactivate QM/Health Check | NEVER leave active on production |
